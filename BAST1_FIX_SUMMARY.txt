TESTING CHECKLIST FOR USER:
================================================================================

Before Testing:
✓ Make sure you are logged in to the application
✓ Navigate to BAST 1 page

Testing Steps:
1. Click "Tambah Data BAST" button
   → Modal should open

2. Select "Nama PT" from dropdown
   → No Kontrak dropdown should filter

3. Select "No Kontrak"
   → Pekerjaan field should auto-fill
   → Tanggal Terima BAST might show related date

4. Fill all required fields:
   - Tanggal Terima BAST (required)
   - Keterangan (required)
   - Opsi Retensi (required)
   - Optional: Tanggal Pusat, Tanggal Kontraktor, PDF file

5. Click "Submit" button
   → AJAX will process the form

6. Check browser console (Press F12, go to Console tab)
   → You should see logs showing:
      "Form #addBastForm submit triggered!"
      "Form data: { ... }"
      "Sending AJAX to: ..."

7. Check for response:
   ✓ Success: Alert "Data berhasil disimpan!" → Page reloads
   ✓ Error: Alert with specific error message

Expected Results:
- If successful: New BAST data appears in table immediately after reload
- If error: Clear error message indicates what went wrong (validation, file, etc.)

Troubleshooting:
- Error "Data berhasil disimpan!" but no new data in table?
  → Page reloaded but data not visible - refresh page manually
  
- Error "404 URL tidak ditemukan"?
  → Route not recognized - check routes.php has new route
  
- Error "403 Akses ditolak"?
  → Session/login issue - logout and login again
  
- Error "500 Server error"?
  → Check application logs at application/logs/log-*.php
  
- Form appears to submit but nothing happens?
  → Check browser console (F12) for AJAX errors
  → Check network tab to see actual request/response
